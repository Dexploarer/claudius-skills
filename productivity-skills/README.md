## üéØ Philosophy

### Why This Exists

The Claudius Skills repository focuses on coding and development - but **productivity is universal**. Whether you're a:
- Software engineer managing projects
- Creative professional juggling clients
- Team leader coordinating people
- Individual trying to achieve personal goals

...you need productivity systems that work.

### Core Principles

1. **AI as Thought Partner** - Not just task executor, but co-thinker
2. **Systems > Motivation** - Build reliable systems, don't rely on willpower
3. **Action > Planning** - Plans must drive execution, not replace it
4. **Simplicity > Complexity** - Use what works, ignore the rest
5. **Continuous Improvement** - Iterate on your productivity system

### What This Is NOT

- ‚ùå A replacement for doing the work
- ‚ùå A productivity religion (use what works, skip the rest)
- ‚ùå One-size-fits-all (customize to your needs)
- ‚ùå Set-it-and-forget-it (requires active engagement)

**Remember:** Tools enable action. You must take action.

## üÜò Troubleshooting

**Not seeing results?**
- Are you executing on plans you create?
- Are you using tools consistently (weekly review, journaling)?
- Have you customized tools to your workflow?
- Are you being honest in reflections?

**Feeling overwhelmed?**
- Start with just 3 tools (email, task-breakdown, weekly-review)
- Use them for 2 weeks before adding more
- Quality over quantity

**Skills not triggering?**
- Try different trigger phrases (check skill files)
- Be more explicit: "Use the email-composer skill to..."

**Want to add new capabilities?**
- Check `/templates/` for creating skills/commands
- Browse `/examples/` for patterns
- Customize existing tools first before building new ones
